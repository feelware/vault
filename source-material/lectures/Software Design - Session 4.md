# Software Design - Session 5

![Fuentes de requerimientos](utilities/attachments/Pasted%20image%2020240921081929.png)

## Software Requirement Specification (SRS)

![componentes del srs](utilities/attachments/Pasted%20image%2020240921083755.png)

### Requisitos

- Cada requisito es una "tarjeta"
- Cada requisito funcional tiene al menos tres campos: descripción, pre-condición y pos-condición ![ejemplo de tarjeta requisito srs](utilities/attachments/Pasted%20image%2020240921084411.png) ![otro ejemplo de tarjeta requisito srs](utilities/attachments/Pasted%20image%2020240921084504.png)
- Cada requisito no funcional (usabilidad, rendimiento, etc.) puede tener uno o más puntos que lo sustenten ![ejemplo con un solo punto de sustentación por requisito no funcional](utilities/attachments/Pasted%20image%2020240921084644.png)

## Matriz de actividades y requerimientos

- lista cada actividad de negocio
- registra si es automatizable o no
	- si no es automatizable, la fila se detiene ahí
	- si sí es automatizable, se registra los requisitos funcionales a los que está asociada (pueden ser varios)
	- cada requisito funcional está asociado a un (y solo un) caso de uso

![ejemplo](utilities/attachments/Pasted%20image%2020240921090816.png)
